# SamFlow

A local, private dictation tool for macOS. **Hold `Fn`, talk, release** — the text
appears in whatever app has focus. Nothing leaves your machine: transcription runs
on-device with [whisper.cpp](https://github.com/ggml-org/whisper.cpp) on the Apple
Silicon GPU.

Think of it as a self-hosted Wispr Flow / macOS Dictation replacement that you fully
control and can teach your own vocabulary.

- **Fast** — the model stays warm in a background service, so an 8-second clip
  transcribes in ~0.5s instead of the ~11s a cold start costs.
- **Private** — no network, no account, no per-word cost.
- **Yours** — a plain-text vocabulary and rule list you extend when it mishears a word.
- **Polished** — a floating pill that follows your cursor, pauses your music while you
  talk, and never degrades your Bluetooth audio quality.

> Requires **macOS on Apple Silicon** (M1 or newer). Defaults to Dutch (`nl`); see
> [Customising](#customising) to switch language.

## What gets installed

`install.sh` sets all of this up for you, but here is the full list so you know exactly
what lands on your machine:

| Component | What | How it's installed | Size |
|---|---|---|---|
| **whisper.cpp** | the transcription engine (`whisper-server`, Metal GPU) | `brew install whisper-cpp` | ~small |
| **ffmpeg** | audio decoding | `brew install ffmpeg` | ~small |
| **uv** | manages an isolated Python 3.12 (survives Homebrew upgrades) | `astral.sh/uv` installer | ~small |
| **Whisper model** | `ggml-large-v3-turbo-q8_0.bin` | downloaded from Hugging Face into `models/` | **~834 MB** |
| **Python packages** | `sounddevice`, `numpy`, `requests`, `pyobjc-*` | `uv pip install -r requirements.txt` into `.venv/` | ~small |
| **Warm-model service** | `com.samflow.server` launchd agent keeping the model loaded | generated by `install.sh` → `~/Library/LaunchAgents/` | — |
| **SamFlow.app** | a tiny app bundle that runs the Fn listener and auto-starts at login | built by `install.sh` → `~/Applications/` | tiny |

The `.venv/` and the model file are git-ignored — they are downloaded locally, not
committed.

## Install

```bash
git clone <this-repo> samflow
cd samflow
./install.sh
```

`install.sh` is safe to re-run; each step checks whether it is already done. It installs
the dependencies above, downloads the model, builds `SamFlow.app`, and starts the warm
service. Then it prints the manual steps below — macOS requires these to be done by hand
in the GUI:

**1. Grant permissions** (microphone, Input Monitoring, Accessibility):

```bash
open -n -a ~/Applications/SamFlow.app --args --grant
```

Click **Allow** on the microphone prompt, then switch **SamFlow** on in
*System Settings → Privacy & Security → Accessibility* and *→ Input Monitoring*.

Why an `.app` and not just a script: macOS attaches these permissions to an app's
stable identity. A bare Python binary loses them the moment its path changes (which
`uv` upgrades do). The bundle fixes that permanently.

**2. Free the Fn key:** *System Settings → Keyboard → "Press 🌐 key to" → "Do Nothing"*,
otherwise macOS pops the emoji picker every time you start dictating.

**3. Verify:**

```bash
open -a ~/Applications/SamFlow.app
./.venv/bin/python samflow.py --check     # all green?
```

## Usage

Hold **Fn**, speak, release. A pill appears near your cursor with live audio bars, then
the text is pasted where you were typing. A menu-bar dot (`◌`) shows status; click it to
see state or quit.

```bash
./.venv/bin/python samflow.py           # run in the foreground (dev)
./.venv/bin/python samflow.py --once     # record one clip, print it, don't paste
./.venv/bin/python samflow.py --check    # permissions, mic, server
./.venv/bin/python samflow.py --review   # promote words you often say into your lexicon
python cleanup.py                        # run the text rules against their examples
```

Once installed, SamFlow runs automatically at login — you never start it by hand.

## Customising

SamFlow learns *your* words through one file, `lexicon.txt` (personal, git-ignored):

- **Your terms** — one per line, in the right spelling (`MarketOS`, `sam-os`). They're fed
  to the decoder as a prompt *before* it guesses, and afterwards `lexicon.canonicalise`
  snaps every variant back (`market os`, `marketos` → `MarketOS`) — casing and splits, for
  free. A new line takes effect on the next dictation, no restart. It never touches a word
  that isn't in your list.
- **The easy way:** `samflow.py --review`. SamFlow counts the unknown words you say often
  and offers them; you pick *add as a term* or *map a mishearing* (`klavijo = Klaviyo`,
  stored in `mappings.txt`). That is the whole tuning loop — mostly automatic.

Built-in defaults live in `lexicon.py` (`DEFAULT_TERMS`); rare non-personal phonetic misses
live in `REPLACEMENTS` in `cleanup.py`.

**Language:** set `LANGUAGE` in `samflow.py` (`"nl"`, `"en"`, …) and re-run `install.sh`
so the warm service matches. Put terms in your language in `lexicon.txt`.

Other knobs (all near the top of their file): `SILENCE_RMS`, `PREROLL_SEC`,
`IDLE_CLOSE_SEC` in `samflow.py`; `WARN`/model choice; `MIC_DEVICE` in `audiodev.py`;
`PAUSE_MEDIA`, `FOLLOW_CARET`.

## How it works

```
Fn down ─► mic ─► Fn up ─► whisper-server ─► cleanup.py ─► Cmd+V
          (+0.4s              (warm, ~0.5s)   (vocab +      (clipboard
           pre-roll)                           rules)        borrow & restore)
```

| File | Responsibility |
|---|---|
| `samflow.py` | the daemon: Fn event-tap, microphone, server client, pasting |
| `lexicon.py` | your word list, the corrector that snaps variants back, and the learn-loop |
| `cleanup.py` | deterministic text rules: fillers, stutters, hallucinations, capitalisation |
| `hud.py` | the floating pill + menu-bar item |
| `focus.py` | where is the user typing? caret → mouse-in-window → window |
| `media.py` | pause Spotify/video while you hold Fn, resume after |
| `audiodev.py` | pick the mic; avoid Bluetooth (keeps AirPods on full audio quality) |
| `macos/`, `install.sh` | the app bundle + launchd templates and the installer |

### Why whisper.cpp and not faster-whisper

whisper.cpp uses the Apple Silicon GPU (Metal) and `mmap`s the model from disk, so its
pages are file-backed and evictable rather than competing for RAM/swap. faster-whisper
(CTranslate2) is CPU-only on Apple Silicon and holds the model in the Python heap. For
push-to-talk on a memory-constrained machine, whisper.cpp is the right trade.

### Sharp edges (each was once a bug)

- **Whisper invents text from silence.** Fed 2s of silence it returned `Www.Nil.Com.Br`.
  An energy gate in `handle()` drops anything below `SILENCE_RMS` before it reaches the
  server; a hallucination list is the backstop.
- **A denied microphone raises no error on macOS** — just a stream of zeros. `--check`
  asks AVFoundation directly instead of trusting that the stream opened.
- **Recording from a Bluetooth mic wrecks its output audio.** Opening the AirPods mic
  forces the codec from 48 kHz (music) to 24 kHz (telephone). `audiodev.py` records from
  the built-in mic instead, so your music stays clean.
- **Whisper sometimes splits a word with a newline** (`git\nhub`). A real segment
  boundary always has a space after the newline; `cleanup._join_segments` uses that to
  tell them apart.
- **The pill must never take focus** — it's a non-activating `NSPanel`. If it did, the
  `Cmd+V` would paste into the pill instead of your editor.
- **The uv-managed Python is deliberate.** A Homebrew venv resolves to a path with the
  patch version baked in; one `brew upgrade` breaks the venv *and* its permissions. uv's
  interpreter links via `@rpath` and lives in your home, so it only moves when you move it.

## Uninstall

```bash
launchctl bootout gui/$UID/com.samflow.server
rm -f ~/Library/LaunchAgents/com.samflow.server.plist
rm -rf ~/Applications/SamFlow.app
# then remove "SamFlow" from System Settings → General → Login Items
```
